## 2. 좌표계 / 데이터 규칙 (★중요)

- `S(x,y) = [x, -y]` — 화면좌표 변환(북쪽 위)
- 측점: `p.x`, `p.y` (코드 내부값), `p.no = "YYMMDD-N"` (예 `260522-12`)
- **좌표계 = 세계측지계 중부원점(GRS80) = EPSG:5186**. CSV는 (북, 동) 순서로 들어옴(예 `517281`=북, `202592`=동). 카카오맵 연동 시 (동,북) 순서로 proj4에 투입.
- **코드(관공 정보) 해석**: 예 `FC 100 x 3`
  - `FC` = 관종(FC관/COD관), `100` = 관경 100mm, `3` = **관수(매설 관 개수)** ← 자동결선 핵심
- 판별 헬퍼 (index.html):
  - `pipeCount(p)` 정규식 `/[xX×]\s*(\d+)/`
  - `isTpoint(p)` 정규식 `/(^|\s)T(\s|$)/` (예 `FC T 100x5`)
  - `isManhole(p)` 정규식 `/^\s*M/` (예 `M FC 100x5`)
- **M(맨홀)** = 별도 구조물이 아니라 "맨홀 바로 앞"의 **일반 관로 측량점** → 결선·사진 이웃에 정상 포함. 시각적 이중원(◎)만 추가.
- **T(분기점)** = 한 줄로 오다 2·3·4가닥 분기/합류. **T점 관수 = 합류 가닥 관수 합** (예 11번 3관 + 9번 2관 = 12번 T점 5관 → 13번 맨홀 5관).

## 4. 구현된 기능 (전부 동작)

- **기초자료/저장**: CSV(EUC-KR/UTF-8 자동), 결선 DXF, 현장 저장/불러오기. 저장 payload = `points/lines/labelOff(측점 라벨 위치)/markups`. `lines`에 지거 멘트(`note`)·위치(`noteOff`) 보존.
- **툴바**: 좌측 카테고리 레일 + 상단 컨텍스트 서브바. `var TB=[...]` 데이터 구동. 카테고리 = 기초자료/선택·이동/결선/관로선편집/맨홀편집(soon)/압입구간편집(soon)/검수. 테마3(흰 배경+좌측 색띠).
- **자동결선 `autoConnect()` (관수 연속성 기반)**:
  - A) 같은 관수끼리만 nearest-neighbor 체인(THRESH=40). 관수 다르면 안 이음 → 없으면 그 점에서 끝.
  - B) T점 합류: 주위 더 작은 관수 점들을 가까운 순으로 합이 T관수가 될 때까지 연결.
  - 각 연결을 2점 통신관로 라인으로 등록. 맨홀도 관수 따라 포함.
- **그리기 엔진**: `startDraw(layer)` 라이브 고무줄, Enter/Space로 완료, 되돌리기=한 점 취소, 점 근처 파란 스냅. 관로선=빨강, **지거선=노랑 굵게(w2.4)**. 지우기는 hover 굵게 표시 후 클릭(`deleteSegmentAt`,`delLayer`).
- **지거 멘트**: 지거선 중심 고정점 + 인출선 라벨, **드래그 이동**(`addNoteHandle`, `noteOff` 저장), 점선 박스.
- **측점 표시**: **속 빈 두꺼운 사각 테두리**(stroke-width 7.2 non-scaling, 가운데 투명 → 라인이 중심까지 보임). **T점=진한 초록(#13803a)**, 일반=검정(#111). (옛 stub 제거됨)
- **사진 패널**: 선택측점/위·아래 측점. `neighborsOf`(결선 있으면 `lineNeighbors`, 없으면 nearest 위/아래). M점도 이웃 포함. CSV 측점번호 끝자리↔사진 파일명. 연동/미연동 토글, 확대/이동, 사진 업로드/삭제.
- **좌표 표시**: 우하단 화면 고정(`placeCoord`), 줌·로드·패널열기에도 유지.
- **검수 마크업**: 박스·써클·확인(초록)·이상(빨강) + 검수기록 패널. 되돌리기/심벌지우기/보기.
- **LINECOL**: 통신관로 `#d92b2b` w1.6 / 압입구간 `#1f6fd6` w1.6 dash / 지거 `#f2b400` w2.4 / 주입상인출선 `#999`.

## 7. 다음 작업 — 카카오맵 백판 + 로드뷰 (★진행 예정)

직원 요청: 도면창에 **수치지도(카카오맵) 백판**을 깔고, **클릭하면 로드뷰가 그 위치로 자동 연결**되게.

### 진행 순서 (이 순서대로)
1. **좌표 변환 + 검증 (가장 먼저, 별도 테스트 html)** — proj4js로 측량좌표 → 위경도. 측점 몇 개를 카카오맵에 찍어 **실제 위치 일치 확인**. (동/북 컬럼 순서도 여기서 확정) ← 이게 통과돼야 다음 단계 의미 있음.
2. **카카오맵 백판 + 측점·결선 오버레이** — 검증되면 도면창에 지도 깔기. 구조 결정: (a) 지도 베이스 + 카카오 오버레이로 재렌더(깔끔) vs (b) SVG를 지도 위에 얹어 줌/이동 동기화.
3. **로드뷰 연결** — 측점/지점 클릭 → 그 위경도 로드뷰(`kakao.maps.RoadviewClient` + `Roadview`).

> 지도 타입은 일반지도/스카이뷰(위성)/하이브리드 선택 가능. 백판은 보통 스카이뷰 또는 일반지도로.

### 좌표계 원점별 proj4 정의 (GRS80, x_0=200000, y_0=600000 공통, lon_0만 다름)
```
서부 EPSG:5185  +proj=tmerc +lat_0=38 +lon_0=125 +k=1 +x_0=200000 +y_0=600000 +ellps=GRS80 +units=m +no_defs
중부 EPSG:5186  +proj=tmerc +lat_0=38 +lon_0=127 +k=1 +x_0=200000 +y_0=600000 +ellps=GRS80 +units=m +no_defs   ← 현재 현장
동부 EPSG:5187  +proj=tmerc +lat_0=38 +lon_0=129 +k=1 +x_0=200000 +y_0=600000 +ellps=GRS80 +units=m +no_defs
동해 EPSG:5188  +proj=tmerc +lat_0=38 +lon_0=131 +k=1 +x_0=200000 +y_0=600000 +ellps=GRS80 +units=m +no_defs
```
- 도면창에 **원점 선택(중부/동부/서부/동해)** 드롭다운 추가 예정 (현장마다 다를 수 있음).
- 변환 시 proj4에는 **(easting=동, northing=북)** 순서로 투입.
